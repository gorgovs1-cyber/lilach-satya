הצהרת נגישות

עודכן לאחרונה: יוני 2026

אתר זה, המופעל על ידי לילך חזן – מאמנת להתפתחות אישית בשיטת סאטיה, פועל מתוך מחויבות לנגישות דיגיטלית לכלל המשתמשים, לרבות אנשים עם מוגבלויות. אנו שואפים לעמוד בדרישות תקן WCAG 2.1 ברמה AA ובדרישות תקנות הנגישות הישראליות.

מה נעשה לשיפור הנגישות

  • קישור "דלג לתוכן הראשי" לניווט מהיר עבור משתמשי מקלדת
  • הגדרת שפת הדף (lang="he") וכיוון RTL לתמיכה בקוראי מסך
  • שימוש בתגיות סמנטיות: <main>, <nav>, <footer>
  • טקסט חלופי (alt) מתאר לכל התמונות
  • ניגודיות צבעים של 4.5:1 לפחות לכל הטקסטים הרגילים
  • מסגרת focus ברורה לכל הרכיבים האינטראקטיביים
  • תמיכה מלאה בניווט מקלדת (Tab, Enter, Space, Esc, מקשי חצים)
  • מלכוד פוקוס (focus trap) בתפריט הנייד וסגירה במקש Esc
  • ARIA מלא על רכיבי אקורדיון (שאלות נפוצות) וטאבים (אבני היסוד)
  • כותרות בסדר היררכי תקין (H1 ← H2 ← H3)
  • כיבוד העדפת prefers-reduced-motion לצמצום אנימציות
  • סימון אלמנטים דקורטיביים כ-aria-hidden להסתרה מקוראי מסך

מגבלות נגישות ידועות

אנו פועלים לשיפור מתמיד של הנגישות. ייתכן שרכיבים מסוימים טרם הונגשו במלואם. אם נתקלת ברכיב שאינו נגיש, נשמח שתעדכן/י אותנו ונפעל לתקנו בהקדם.

תאימות לדפדפנים ולטכנולוגיות עזר

האתר נבדק ונתמך ב:

  • Chrome, Firefox, Safari – גרסאות עדכניות
  • ניווט מקלדת מלא
  • קורא מסך NVDA (Windows) ו-VoiceOver (Mac/iOS)

בסיס חוקי

Accessibility Statement

Last updated: June 2026

This website, operated by Lilach Hazan – personal development coach using the Satya Method, is committed to digital accessibility for all users, including people with disabilities. We strive to meet the requirements of WCAG 2.1 Level AA and the Israeli accessibility regulations.

What We Have Done

  • A "Skip to main content" link for quick keyboard navigation
  • Page language (lang="he") and RTL direction for screen reader support
  • Semantic tags: <main>, <nav>, <footer>
  • Descriptive alternative text (alt) for all images
  • Color contrast of at least 4.5:1 for all normal text
  • Clear focus outlines on all interactive elements
  • Full keyboard navigation (Tab, Enter, Space, Esc, arrow keys)
  • Focus trap in the mobile menu and closing with the Esc key
  • Full ARIA on accordion (FAQ) and tab (foundations) components
  • Proper heading hierarchy (H1 → H2 → H3)
  • Respect for the prefers-reduced-motion preference
  • Decorative elements marked aria-hidden to hide them from screen readers

Known Limitations

We are continuously working to improve accessibility. Some components may not yet be fully accessible. If you encounter an inaccessible element, please let us know and we will work to fix it promptly.

Browser and Assistive Technology Compatibility

The site has been tested and is supported on:

  • Chrome, Firefox, Safari – latest versions
  • Full keyboard navigation
  • NVDA (Windows) and VoiceOver (Mac/iOS) screen readers

Legal Basis

This statement is provided in accordance with the Equal Rights for Persons with Disabilities (Service Accessibility Adjustments) Regulations, 2013 and Israeli Standard 5568, based on WCAG 2.1.
